### 2.1 全域設定：倒角快速切換（`C0.2`／`C0.3`／不指定）與「僅倒角模板」

以下為**產品已定義語意**（實作錨點含 **`palette.html`** 與對應資料流）；補足 **§1／§2** 之概括，並避免將**不同規格**綁在同一輪改動中。

#### `C0.2`／`C0.3`／不指定倒角

- **用途**：快速切換模板所對應之**倒角大小**（`C0.2`、`C0.3`、不指定倒角）。

#### 「僅倒角模板」

- **主要用途**：翻面加工後，為**快速對工件倒角**而使用；於 UI 上用以**屏蔽（過濾）一般加工模板**，使列上僅保留與倒角套用相關之選項與操作語境。

#### 兩者結合

- **合併語意**：兩者結合即 **「快速切換倒角規格」** 與 **「模板套用篩選」** 之一致功能；變更任一半之互斥、觸發範圍或與下拉／列狀態之銜接前，須依 **§1**。

#### 現況與擴充（禁止擅自更動契約）

- **現階段套用範圍**：**孔加工區塊**與**長條孔加工區塊**。
- **未來擴充**：若加入**外輪廓**、**內輪廓**等特徵辨識，**模板辨識＋倒角功能篩選**預期仍沿用**同一套**倒角快速切換與「僅倒角模板」之**全域語意**；故本小節所定行為**不得擅自更改**（含改為非全域、破壞互斥、或與「屏蔽加工模板／快速倒角」目的相違之重寫），**須 §1** 並同步更新本節與相關設計文件。

#### 實作與 scope 紀律（與上列並讀）

- **分開視為兩項需求**：僅使用者點名其中一項時，**不得**順手改寫另一項之語意或觸發條件。

## 4. 鑽孔工序與模板語義（核心）

- **整數版 `V1.0`「辨識→模板工序→刀路」管線之事實**：在 **CAM 鑽孔端**，**§4.1** 所述 **`holeFaces` 參數鏈寫入**為該管線賴以成立之**核心奠基**；變更 **§4.1** 所依源碼語意前須 **§1** 並全回歸。  
- **鑽孔工序（主鑽、同徑展開、孔面綁定等）屬核心行為**，非經**明確同意**不得改寫語意。  
- **同直徑**：以**一個代表孔面**讓模板以**相同直徑自動加工**；實作須符合「減少不必要計算」的意圖，**不得**用與需求相反的手段（例如錯誤關閉主流程應有的同徑行為）。  
- 與 **§3** 並讀：通／盲、沉頭、合併等仍受 **權威判定鏈** 約束。

### 4.1 Fusion CAM `holeFaces` 幾何寫入（與主檔源碼一致）

以下為**已驗證產品邏輯**之摘要；**權威實作**以主檔 **`_append_seed_faces_to_existing_drill_op`**（及與其一致之綁孔／併孔段落）為準。Autodesk 公開文件對此鏈路著墨不足，**不得以未經驗證之 MCP／對話結論**覆寫源碼語意；變更須 **§1** 並回歸。

- **文件與版號**：本節**不**逐版登記 Fusion 發行號；維護者**以本倉庫源碼與回歸為準**即可。官方若未公開此參數鏈路，外部問答亦無從取代實測結論，**無須**在本準則追蹤「何時官方有寫、AI 是否知悉」。

- **語意定位（效果類比）**：在 **`holeMode = 'selection-faces'`** 之前提下，本節所述 **`holeFaces` 參數鏈寫入**所達成者，是**觸發與使用者在工序編輯中「以面選孔」相近之 CAM 幾何綁定效果**（**類比**說明，**不**宣稱與滑鼠事件在實作層級逐項等同）。

- **適用範圍**：**僅** **CAM 鑽孔類工序**之 **`holeFaces`**；維護者以此參數鏈寫法**穩定帶入 BRep 孔面**。**不**推廣為其他工序類型之一般命題。

- **穩定性預期（綁定層）**：此行為依 **Fusion Python API 所暴露之 CAM 參數鏈**（底層為 **C++／SWIG 綁定**之 getter／setter 語意）；**非**一般純 Python 邏輯，**在 Fusion 未對 CAM 參數鏈做大破壞性改版之前**，預期可長期沿用。若遇大改版導致鏈路失效，依 **§1** 重驗、回歸後再修本節與源碼。

- **與整體版次之關係（整數 `V1.0` 之核心奠基）**：**整數版 `V1.0` 所代表之「辨識→模板工序→刀路」管線**，在 **CAM 鑽孔端**賴以成立之**核心能力之一**，即本節 **`holeFaces` 參數鏈寫入**（缺則無法於 **`selection-faces`** 模式下**穩定**把辨識所得 **BRep 孔面**綁上工序）。其可運作前提尚包含：在**常規 Fusion（無依賴未於本專案驗收之擴充能力）**下，上開綁定能**可靠**完成；滿足此前提之**已驗證**作法即本節與主檔一致之寫法。公開文件**未**載明「直接餵入孔特徵幾何模型」之完整腳本路徑時，**於本專案鑽孔工序綁定需求內**，本寫法為**目前所知唯一能穩定達成該綁定**之路徑；若有替代 API，須**經 §1 核准與回歸**後再更新本節。

- **與原廠「孔辨識」策略之差異（產品定案，非通用評測）**：Fusion 產品內建之孔相關流程，常見為以**建立孔特徵／定型孔幾何**為主軸之**產品級定型策略**（便於一般建模與編輯一致性）。本專案在**將幾何帶入 CAM 鑽孔工序**時，改以 **§3** 權威判定鏈（通／盲、沉頭、合併、射線輔助等）辨識後，再以 **BRep 孔面**綁定 **`selection-faces`**；於**「工序面指派與加工語意可控性」**之已定案需求上，**較貼合本外掛驗收場景**。**不**宣稱於所有使用者情境全面優於原廠策略；比較範圍**僅限**本專案 CAM 鑽孔綁定路徑。

- **核心寫法（參數鏈）**：自 **`faceParam = params.itemByName('holeFaces')`** 取得 **`vec = faceParam.value.value`**（**Fusion CAM 參數所暴露之可變更集合 proxy**）；就地 **`vec.push_back(face)`**；必要時依主檔備援 **`vec.add(face)`**；若有新增，再 **`faceParam.value.value = vec`** 沿**同一條** `value.value` property chain 寫回。**禁止**以 **`CadObjectParameterValue.cast(faceParam.value)`** 等**旁路物件**再對 `.value` 賦值（易與參數樹脫勾、寫回不生效）。**禁止**將 **Python `list`** 或未經該參數鏈取得之**外來集合**直接塞給 `.value`（型別／語意不符）。

- **執行順序（以源碼為唯一權威）**：主檔為**先**以 **`_set_exp`**（或等效）將 **`holeMode`** 設為 **`'selection-faces'`**，**再**取得 **`vec`**、去重、**`push_back`／`add`**、寫回。「先／後」是否在其他工序狀態下更穩，屬**實測議題**；**預設與變更準則一律跟倉庫主檔**，**不以**聊天或單次腳本實驗覆蓋。若實驗主張改順序：須**另分支**、**全回歸**、合併後再更新本節與 **`docs/版本紀錄.md`**。

- **名詞**：不將該集合記述為 **`std::vector`** 等未經佐證之底層型別；統稱 **「參數專屬可變更集合 proxy」** 即可。

- **錯誤處理**：新寫或摘錄之範例**不得**使用 **`except: pass`** 隱匿失敗；須與主檔風格一致（**辨識失敗原因、回傳可判定之成功／失敗**）。

## 5. 大孔通孔預鑽 — 產品定案摘要（與程式對齊）

- 僅適用 **通孔**、孔徑 **嚴格大於** **`LARGE_HOLE_PILOT_THRESHOLD_MM`（6.5）mm**（**不含** 等於 6.5）之列；且程式另排除 **沉頭大孔列**（`isCBLarge`，與 `_executeFromPalette` 條件一致）。  
- 從 **2.6～5.5 mm**（`PILOT_DRILL_DIAMETER_MIN_MM`～`PILOT_DRILL_DIAMETER_MAX_MM`）鑽頭範圍內，於已登錄之既有工序選 **直徑最大** 的 **drill**。  
- 將 **大孔之孔面（同一徑一個種子面）** 併入該道既有鑽孔工序（`holeFaces` + **`selectSameDiameter`**），**不**另建預鑽專用模板工序。  
- **只影響鑽頭**；**不得**擅自改寫中心鑽、倒角、絞刀等非鑽頭路徑。  
- **工序排序概念**：表格上大孔列在後，使處理大孔時小徑鑽已建立並可登錄於 `pre_drill_registry`（與 `docs/開發對話與變更.md` §1 一致）。  
- **主鑽新建**：`_createOpFromTemplate(..., select_same_diameter=True)` 仍由模板自動展開同徑；與「種子面只傳一個」並存（見主檔註解與 `faces_one_seed`）。

### 8.2 孔用倒角模板（鍵 `holeChamfer`；與 `contourChamfer` 分開）

- **路徑鍵**：主孔表 **`chamferMap`** 在**嚴格載入**（已自 **`generalHole`／`tapHole`／`locatingHole`** 任一取得鑽模板）時，**只**自 **`TEMPLATE_FOLDER_PATHS["holeChamfer"]`** 所定之階層蒐集：

  `%AppData%\Roaming\Autodesk\CAM360\templates\AL6061\倒角刀模塊 【AL6061】\孔倒角 【AL6061】`  
  **`S50C`** 將 **`AL6061`** 全替為 **`S50C`**。

### 8.3 孔加工模板與孔倒角之結合（概要）

- **資料流**：材質變更或重掃後，**`buildTemplateMaps`** 建 **`drillMap`**／**`chamferMap`**；**`buildTable`** 依列孔徑呼叫 **`buildDropItems`**，產生該列之下拉項（每項最多含 **`drillUrl`** 與 **`chamferUrl`** 各一，可為空）。  
- **執行**：選項帶入之 URL 決定是否自樣板庫建立鑽／鉸／搪與／或倒角工序；**鑽與倒角可只選其一**。  
- **槽孔／沉頭**：槽孔列與主孔表**不同池**；沉頭大孔列在 UI 上走 **`buildDropItems`** 之 **`isCBLarge`** 分支（與一般列篩選不同）。  
- **沉頭孔鑽模板之路徑鍵**：**`countersinkHole`**，階層與 **§8.1** 表及 **`TEMPLATE_FOLDER_PATHS`** 一致。本機 **`CAM360\templates`** 磁碟常見對照例（**`S50C`**）：  
  `%AppData%\Roaming\Autodesk\CAM360\templates\S50C\孔加工模塊 【S50C】\沉頭孔 【S50C】`  
  （**`AL6061`** 則將路徑中之 **`S50C`** 全替為 **`AL6061`**；與 **§8.2**「材質一對一替換」規則相同。）  
- **沉頭列之倒角**：與一般孔、主孔表**相同**，仍取自 **`chamferMap`**（路徑鍵 **`holeChamfer`** → **`{material}/倒角刀模塊 【{material}】/孔倒角 【{material}】`**），**套用孔倒角**；**無**另行獨立之「沉頭專用倒角」樣板路徑鍵。  
- **完整且可核對之規格（載入、索引、篩選、組合順序、關鍵字與數值）**：一律以 **§8.4** 為準；**§8.3** 不單獨作為實作依據。

### 8.4 孔加工模板：載入、直徑索引、篩選與下拉結合之完整契約

以下與現行程式**意圖對齊**；若程式碼與本節衝突，視為 **bug 或文件落後**，須由人類決定改程式或改文件（**禁止** AI 未依 **§1** 擅自只改一方）。

#### 8.4.1 程式錨點（變更時必查）

| 區塊 | 檔案 | 符號（函式／常數） |
|------|------|-------------------|
| 路徑表 | `半自動加工選單【UI穩定版】.py` | **`TEMPLATE_FOLDER_PATHS`** |
| 倒角白名單字串 | 同上 | **`ALLOWED_CHAMFER_TAGS`**（現為 **`('C0.2', 'C0.3')`**） |
| 載入鑽＋倒角映射 | `templates/template_service.py` | **`build_template_maps`**、**`collect_assets_from_folder_path`**、內部 **`add_to_drill_map`**、**`add_to_chamfer_map`**、**`_scan_chamfer_tree_only`**（後備）、**`_scan_template_tree`**（fallback） |
| 本機庫清單索引 | `templates/template_fs_cache.py` | **`get_material_fs_entries`**、**`invalidate_all`**／**`invalidate_material`**；**`url_join_relative_path`**（仍經 **`LocalLibraryLocation.join`**） |
| 映射封裝 | `半自動加工選單【UI穩定版】.py` | **`buildTemplateMaps`** |
| 列上下拉 | 同上 | **`buildDropItems`**、**`_chamfer_items_by_target_dia`**、**`_drill_items_by_target_dia`**、**`_is_allowed_chamfer_name`** |

#### 8.4.2 鑽模板進 **`drillMap`**（嚴格模式為預設路徑）

1. **`build_template_maps`** 先以 **`strict_keys = ("generalHole", "tapHole", "locatingHole")`** **依序**讀 **`TEMPLATE_FOLDER_PATHS`** 對應之相對階層，以 **`collect_assets_from_folder_path`** 遞迴蒐集該資料夾內模板。  
2. **若上述三鍵中任一路徑成功取得至少一筆資產**，則 **`strict_loaded = True`**：僅將這些資產交 **`add_to_drill_map`**（**不**在此路徑掃 **`countersinkHole`**／**`slotHole`**）。  
3. **`add_to_drill_map`**（鑽側進池前篩選與索引）：  
   - **槽孔模板整筆跳過**：若 **`folder_hint`＋檔名 `leaf`＋`tags`＋描述** 組出之字串（轉小寫）經 **`_looks_like_slot_template`** 判定為槽／長條孔類，**不**入池。關鍵字包含 **`長條孔`、`槽孔`、`slot`、`obround`、`racetrack`**（與程式內槽孔常數一致）。  
   - **沉頭標記**：上述標記字串中若含 **`沉頭`**（`KW_COUNTERSINK`）或 **`countersink`**，則 **`isCountersinkTemplate=True`**（供 **`buildDropItems`** 區分一般列／沉頭大孔列）。  
   - **絞／鑽／搪語意**（供 UI **`hasReamer`／`hasDrill`／`hasMillBore`**）：由模板描述、`tags`、顯示名組合字串比對中文／英文關鍵字（絞孔／絞刀／定位孔／精孔／铰孔；鑽孔／點孔／鑽頭／钻孔；孔銑／擴孔／镗孔 等——**以原始碼字串為準**）。  
   - **牙孔**：若 **`tap_dia_map`** 中某牙規字串出現在 **`leaf`**，則在該牙之 **[最小直徑, 最大直徑]** 範圍內以 **0.1 mm** 步進展開，將同一模板重複掛入多個直徑鍵（與現行 `while round(d,1) <= round(dia_max,1)` 一致）。  
   - **非牙、但檔名含 `D`**：嘗試自 **`leaf`** 解析 **`D` 後第一段數字** 作為直徑鍵（**四捨五入至小數一位**）。  
4. **若三鍵皆無任何資產**（**`strict_loaded=False`**）：退回 **`_scan_template_tree`**，在樣板庫 **`{material}`** 根下分別進入 **`孔加工模塊 【{material}】`**（鑽）與 **`倒角刀模塊 【{material}】`**（倒角）做**相容舊庫**之掃描（與 **§8.2**「向後相容」一致）。  

#### 8.4.3 孔倒角模板進 **`chamferMap`**

1. **當 `strict_loaded` 且 `template_folder_paths` 含非空 `holeChamfer`**：在鑽映射已由 **§8.4.2** 建立後，以 **`collect_assets_from_folder_path(tmpl_lib, material, template_folder_paths["holeChamfer"])`** 蒐集模板，逐筆呼叫 **`add_to_chamfer_map(url, rawName, tags)`**。**不**掃描 **`孔倒角 【{material}】`** 以外之 **`倒角刀模塊 【{material}】`** 子樹（與產品定案：避免多資料夾同名 **`D*`** 模板無法區分）。  
2. **`strict_loaded` 但無 `holeChamfer` 路徑字串**：後備為 **`_scan_chamfer_tree_only`**（整個 **`倒角刀模塊 【{material}】`** 樹）。  
3. **`add_to_chamfer_map`**：**同鑽側**先排除槽孔類模板；自 **`tags` 再至 `leaf`** 尋找 **`D數字`** 正則以決定直徑鍵（小數一位）；牙規則與鑽側類似可展開至多直徑鍵。  
4. **`contourChamfer` 鍵之路徑**僅供 **`load_slot_chamfer_templates`**，**不**經本小節之 **`chamferMap`** 嚴格路徑寫入（見 **§8.2**）。  
5. **`strict_loaded=False`**：仍由 **`_scan_template_tree`** 於 **`倒角刀模塊 【{material}】`** 整樹寫入倒角（舊庫相容）。

#### 8.4.4 列上直徑對表（`dia` 字串 → 池項目）

- **`_drill_items_by_target_dia`** 與 **`_chamfer_items_by_target_dia`** 行為對齊：先將 **`dia`** 轉成**小數一位**之 **`target`**；若 **`str(target)`** 恰為 **`drillMap`／`chamferMap`** 的鍵，則回傳該鍵之列表。  
- **否則**：在所有可轉成浮點數的鍵中，取 **與 `target` 差值最小** 者；**僅當**該最小差 **≤ 0.2（mm）** 時回傳其列表，否則回傳空列表。**不得**在未核准下改動 **`0.2`** 容差（會直接改變可選模板集合）。

#### 8.4.5 倒角名下拉白名單（`chamferMap` 已入池後）

- **`_is_allowed_chamfer_name`**：將顯示名轉**大寫**並**去除空白**後，須**至少包含** **`ALLOWED_CHAMFER_TAGS`** 之一子字串（現為 **`C0.2`** 或 **`C0.3`**）才允許進入 **`buildDropItems`** 之 **`chamferItems`**。  
- **變更允許之倒角標記**＝變更產品可見倒角種類，**須 §1** 並同步本節與版本紀錄。

#### 8.4.6 **`buildDropItems`** 輸出順序與分支（**順序為契約**）

**共同前處理**：  
`drillItems = drillMap.get(dia, [])`；**`base_chamfer = _chamfer_items_by_target_dia(chamferMap, dia)`**，若空則 **`chamferMap.get(dia, [])`**；**`chamferItems =`** 上述經 **`_is_allowed_chamfer_name`** 過濾後之列表。

**A. 一般列（`holeInfo.isCBLarge` 為假）**  
1. 先從 **`drillItems`** **剔除**：**`isCountersinkTemplate`** 為真，或名稱（小寫）含 **`沉頭`** 或 **`countersink`**。  
2. **依序追加**（不得在未核准下對調順序）：  
   - 對**篩後**每個 **`d`** × **`chamferItems`** 每個 **`c`**：**`label = d名 + ' + ' + c名`**，**`drillUrl`／`chamferUrl` 皆有**，**`hasChamfer=True`**。  
   - 對 **`chamferItems`** 每個 **`c`**：**僅倒角**（`drillUrl=None`）。  
   - 對**篩後**每個 **`d`**：**僅鑽側**（`chamferUrl=None`）。  
   - **`(不使用)`**（兩 URL 皆空）。  

**B. 沉頭大孔列（`holeInfo.isCBLarge` 為真）**  
1. **`target_dia = dia`**（註解定案：沉頭孔徑即匹配直徑）。  
2. **`cs_drill`**：優先 **`_drill_items_by_target_dia(drillMap, target_dia)`** 中取 **`isCountersinkTemplate`** 者；若無，則自**尚未依一般列剔除前**之 **`drillItems`** 取 countersink 模板。  
3. **`source`（倒角）**：優先 **`_chamfer_items_by_target_dia(chamferMap, target_dia)`** 經白名單過濾；若空則退回 **`chamferItems`**。  
4. **依序追加**（不得擅自對調）：**每 `cs_drill` × 每 `source` 組合** → **僅 `cs_drill`** → **僅 `source`** → **`(不使用)`**；**本分支結束即 `return`**（**不**再執行 A 段之雙迴圈）。  

#### 8.4.7 執行面（與下拉契約之銜接）

- 使用者選定之 **`drillUrl`／`chamferUrl`** 與 **`hasDrill`／`hasReamer`／`hasMillBore`／`hasChamfer`** 一併驅動後續工序建立與參數寫入；**`None` URL** 表示該類工序**不**自模板建立。  
- **槽孔**：使用 **`slotHole`**、**`load_slot_chamfer_templates`**（**`contourChamfer`**）與槽孔專用 UI 選項（**`docs/INTEGRATED_HANDOFF.md`** §2 04-25），**不**使用本節 **A** 段之 **`buildDropItems`** 組合語意。

- **模板路徑分流（UI 與邏輯一致）**：一般孔／牙孔／定位孔 與 **沉頭孔／長條孔** 之模板池**嚴格隔離**；長條孔／槽孔模板**不得**再混入一般孔下拉（與上列各鍵專用分工一致）。  
- **槽孔（slot）層級**：孔辨識為**第一層主判定**，slot 為**第二層補充**；**不得**讓 slot 流程**覆蓋**孔主流程語義。  
- **列表可加工範圍（`_buildHoleData` 相關語意）**：**通孔**可 **Z+、Z−** 皆顯示；**盲孔**僅 **Z+**；**背面沉頭**（`Z-(CB)` 或 `Z−` 且沉頭大／小孔標記）**排除**、不可加工。  
- **材質與重掃**：`material_change` 觸發 **`full_rescan()`**、執行前後材質／模板一致性為已定案穩定化路徑，**不得**在未說明下改回「局部快取不刷新」類行為。  
- **驗收**：與本節（模板／槽孔／列表／材質與重掃）直接相關之改動，**須**通過 **`docs/CHECKLIST.md`** **§9～§11**（見 **§2** 同一約定）。

## 11. 視線法（唯讀，可關閉）

- **開關**：`ENABLE_VISION_LAYER`（主檔常數，預設 True）；False 時行為與併入 RayVision 前一致。
- **範圍**：`vision/` 僅產生 `runtime_state.vision_snapshot` 與可選驗證草圖；**不得**改寫 §3.1 孔鏈、§8.4 模板契約或 `execute` 語義。
- **掃描**：孔／槽掃描完成後 `_refresh_vision_snapshot()`，重用 `last_hole_scan_rows_raw` 與 `slotInfoList`（與面板同源）。
- **輪廓（2026-05-20）**：須用 `get_machining_contour_faces_wcs` + `get_complete_outer_contour_edges`；**禁止**退回到僅 `get_top_face()`（槽型雙台面會「只有左邊有線」）。
- **槽草圖**：優先 `loop_edges`；後備膠囊 `_draw_slot_capsule`；**禁止**矩形 `_draw_slot_rects` 包住跑道槽。
- **繪製**：`create_recognition_sketch_from_vision(snap, setup=...)` 須傳使用中 CAM Setup；`draw_recognition_sketch` 在 `notify` 先於 `PaletteActionContext`。
- **熱重載**：**勿**為視線法新增 `PaletteActionContext` 欄位；用 `runtime_state.refresh_vision_snapshot_fn`。
- **詳細**：`docs/VISION_CONTOUR_AND_SKETCH.md`、`docs/VISION_SNAPSHOT_v1.md`。
