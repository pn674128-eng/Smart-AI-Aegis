## 5. 曾討論但未實作的優化（與原因）

以下僅列方向；**目前程式維持不變**，因使用者回饋現況可接受，且優化若未寫清 **「改動 → 可能副作用 → 如何驗證」** 不應貿然套用。


| 方向                            | 可能副作用（摘要）                        |
| ----------------------------- | -------------------------------- |
| 刀路改為僅新建工序集合                   | 舊工序相依、或需全重算時漏算、與 UI「全綠勾」不一致。     |
| 條件式／增量排序                      | 工序樹順序與操作習慣不一致；除錯難度上升。            |
| `Design.isComputeDeferred` 批次 | 依文件類型／Fusion 版本行為不一，CAM 是否受益不確定。 |
| 更廣移除 `getOpToolType`          | 需逐類 `cast`，誤判會導致錯誤綁定或略過工序。       |


**後續約定**：若未來要改，採 **「改什麼 → 預期省什麼 → 可能壞什麼 → 怎麼驗證／怎麼關閉」** 再動手。
